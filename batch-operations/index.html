      JavaScript Data Grid 批量操作

      批量 CRUD 操作，避免不必要的渲染周期，提高网格性能。

      概述

      在 Handsontable 中，每个 CRUD 操作都以render() 结束。在大多数情况下，这被认为是预期行为。表格必须在某个时刻反映出请求的更改。不过，有时您可能会发现这种机制略显过度。

      例如，如果您编写了一个使用多个 CRUD 操作的自定义函数，那么这些 CRUD 操作将为每个 API 方法调用一个render()。您只需要在最后进行一次呈现，就足以反映所有更改。您可以将这些组合操作视为单个操作，让呈现等待它们完成。为此，可以使用 suspend the render 来批量处理这些操作。

      这可以提高应用程序的整体性能。批处理多个操作可以减少渲染次数，因此任何以渲染结束的 API 调用都将受益于这一改进。这样可以减少布局垃圾，减少冻结，并提高响应速度。

      有多种 API 方法可用于暂停，但batch()是最常用的方法。它是一个回调函数，在正文内部提供的所有操作完成后执行render()。使用这种方法是最好的做法，因为它更安全、更易用。您只需将所有要批处理的操作放在闭包中即可。Handsontable 会负责暂停，并在最后执行一次render()。

      下面的代码段展示了一个批处理几个操作的简单示例。三个 API 操作被相继调用。如果不把它们放在批处理回调中，每个操作最后都会执行一次render()。有了批处理功能，您可以跳过两次渲染，在最后以一次渲染结束整个操作。这样做更为理想，而且收益会随着放入批处理batch()中的操作数量的增加而增加。

      // 在一个实例上调用批处理方法
hot.batch(() => {
  // 根据需要运行操作
  hot.alter('insert_row_above', 5, 45);
  hot.setDataAtCell(1, 1, 'x');
  hot.selectCell(0, 0);
  // 渲染会在所有操作完成后立即执行
});

      暂停渲染会带来更好的性能，这在批量执行大量操作时尤为明显。图中显示了相同操作在有批处理（深蓝色列）和无批处理（浅蓝色列）情况下的对比。执行时间的速度随批处理操作的数量而增加。

      TIP

      请注意，还可以使用其他方法来批量操作，但这些方法略显高级，使用时应谨慎。在多次暂停渲染后，如果忘记恢复渲染，可能会出现闪烁、故障或其他视觉变形。将渲染类型的方法与专注于操作的方法混合使用，也会导致一些意想不到的行为。总之，在大多数情况下，使用batch()就足够了，而且使用起来也很安全。

      API methods

      以下 API 方法允许暂停：

      • batch()
      • batchRender()
      • batchExecution()
      • suspendRender() and resumeRender()
      • suspendExecution() and resumeExecution()

      通过使用这些方法，您可以暂停：

      • 渲染
      • 执行
      • 渲染和执行。

      术语 "渲染 "直接指 DOM 渲染，而术语 "执行 "指与 DOM 渲染不同的所有操作。目前，只有索引重新计算允许您推迟进程。

      如果不需要批量异步操作，建议首选以 batch\* 为前缀的方法名称，即 batch()、batchRender() 和batchExecution()。以 suspend\* 为前缀的方法名称（即suspendRender() 和 suspendExecution()）是第二选择。这些方法在需要批量异步操作时非常有用。本质上，它们的工作方式与 batch\*方法相同，但必须手动恢复渲染。

      批量*方法

      批量

      该方法同时发送渲染和其他操作。它具有通用性，如果想在应用程序中批量调用多个 API，它尤其有用。

      hot.batch(() => {
  hot.alter('insert_row_above', 5, 45);
  hot.setDataAtCell(1, 1, 'x');

  const filters = hot.getPlugin('filters');

  filters.addCondition(2, 'contains', ['3']);
  filters.filter();
  hot.getPlugin('columnSorting').sort({ column: 1, sortOrder: 'desc' });
  // 表格缓存将被重新计算，执行回调后将调用一次表格呈现
});

      批量渲染

      batchRender()方法是一个回调函数。将 API 调用放在该方法中可以跳过过多的渲染。执行回调后，表格将被渲染。这种方法不容易出错，因为你不必记住恢复渲染。这种方法的唯一缺点是不支持异步操作。

      hot.batchRender(() => {
  hot.alter('insert_row_above', 5, 45);
  hot.setDataAtCell(1, 1, 'x');
  // 执行回调后，表格将渲染一次
});

      批量执行

      batchExecution()是一个回调函数。通过将 API 调用置于该函数中，可以跳过过多的渲染。执行回调后，表格将被渲染。由于无需记住恢复操作，因此不易出错。这种方法的唯一缺点是不支持异步操作。

      hot.batchExecution(() => {
  const filters = hot.getPlugin('filters');

  filters.addCondition(2, 'contains', ['3']);
  filters.filter();
  hot.getPlugin('columnSorting').sort({ column: 1, sortOrder: 'desc' });
  // 执行回调后，将重新计算一次表缓存
});

      暂停和恢复方法

      suspendRender() and resumeRender()

      要暂停渲染过程，请在要批处理的操作之前调用suspendRender()方法。这是一种手动方法。

      暂停后，使用resumeRender()方法恢复进程。每次 suspendRender() 调用都需要对应一次 resumeRender() 调用。例如，如果您调用了 5 次 suspendRender()，那么也需要调用 5 次 resumeRender()。

      hot.suspendRender(); // 暂停渲染
hot.alter('insert_row_above', 5, 45);
hot.setDataAtCell(1, 1, 'x');
hot.resumeRender(); // 记得恢复渲染

      suspendExecution and resumeExecution

      要暂停渲染过程，可以在要批处理的操作之前调用 suspendExecution() 方法。这是一种手动方法。暂停后，必须记得使用 resumeExecution() 方法恢复进程。

      hot.suspendExecution();
const filters = hot.getPlugin('filters');

filters.addCondition(2, 'contains', ['3']);
filters.filter();
hot.getPlugin('columnSorting').sort({ column: 1, sortOrder: 'desc' });
hot.resumeExecution(); // 它在内部更新缓存

      暂停功能现场演示

      下面的示例展示了batch()方法能在多大程度上缩短渲染时间。两个示例共享相同的数据集和操作。第一个示例显示了使用batch()方法时的耗时。运行第二个示例，查看不使用batch()方法时的渲染时间。

        import Handsontable from 'handsontable';
import 'handsontable/dist/handsontable.full.min.css';

const container = document.querySelector('#example1');
const buttonWithout = document.querySelector('#buttonWithout');
const buttonWith = document.querySelector('#buttonWith');
const output = document.querySelector('#output');

const data1 = [
  [1, 'Gary Nash', 'Speckled trousers', 'S', 1, 'yes'],
  [2, 'Gloria Brown', '100% Stainless sweater', 'M', 2, 'no'],
  [3, 'Ronald Carver', 'Sunny T-shirt', 'S', 1, 'no'],
  [4, 'Samuel Watkins', 'Floppy socks', 'S', 3, 'no'],
  [5, 'Stephanie Huddart', 'Bushy-bush cap', 'XXL', 1, 'no'],
  [6, 'Madeline McGillivray', 'Long skirt', 'L', 1, 'no'],
  [7, 'Jai Moor', 'Happy dress', 'XS', 1, 'no'],
  [8, 'Ben Lower', 'Speckled trousers', 'M', 1, 'no'],
  [9, 'Ali Tunbridge', 'Speckled trousers', 'M', 2, 'no'],
  [10, 'Archie Galvin', 'Regular shades', 'uni', 10, 'no']
];

const data2 = [
  [11, 'Gavin Elle', 'Floppy socks', 'XS', 3, 'yes'],
];

const data3 = [
  [12, 'Gary Erre', 'Happy dress', 'M', 1, 'no'],
  [13, 'Anna Moon', 'Unicorn shades', 'uni', 200, 'no'],
  [14, 'Elise Eli', 'Regular shades', 'uni', 1, 'no']
];

const hot = new Handsontable(container, {
  data: data1,
  width: 'auto',
  height: 'auto',
  colHeaders: ['ID', 'Customer name', 'Product name', 'Size', 'qty', 'Return'],
  licenseKey: 'non-commercial-and-evaluation'
});

const alterTable = () => {
  hot.alter('insert_row_above', 10, 10);
  hot.alter('insert_col_start', 6, 1);
  hot.populateFromArray(10, 0, data2);
  hot.populateFromArray(11, 0, data3);
  hot.setCellMeta(2, 2, 'className', 'green-bg');
  hot.setCellMeta(4, 2, 'className', 'green-bg');
  hot.setCellMeta(5, 2, 'className', 'green-bg');
  hot.setCellMeta(6, 2, 'className', 'green-bg');
  hot.setCellMeta(8, 2, 'className', 'green-bg');
  hot.setCellMeta(9, 2, 'className', 'green-bg');
  hot.setCellMeta(10, 2, 'className', 'green-bg');
  hot.alter('remove_col', 6, 1);
  hot.alter('remove_row', 10, 10);
  hot.setCellMeta(0, 5, 'className', 'red-bg');
  hot.setCellMeta(10, 5, 'className', 'red-bg');
  hot.render(); // 这里需要进行渲染，以填充新的 "className"。
}

let loggedText = '';
let counter = 0;

const logOutput = msg => {
  counter++;
  loggedText = `[${counter}] ${msg}\n${loggedText}`;
  output.innerText = loggedText;
}

buttonWithout.addEventListener('click', () => {
  const t1 = performance.now();
  alterTable();
  const t2 = performance.now();

  logOutput('Time without batch ' + (t2 - t1).toFixed(2) + 'ms');
});

buttonWith.addEventListener('click', () => {
  const t1 = performance.now();
  hot.batch(alterTable);
  const t2 = performance.now();

  logOutput('Time with batch ' + (t2 - t1).toFixed(2) + 'ms');
});
