      概述

      公式插件使用电子表格符号为您提供基于公式的广泛计算功能。在引擎盖下，它使用了一个名为HyperFormula (opens new window)的引擎，该引擎由 Handsontable 团队创建，是一个独立的库，可帮助开发人员构建复杂的数据管理应用程序。

      该插件拥有一个包含 386 个函数的函数库，这些函数分为数学和三角函数、工程函数、统计函数、金融函数和逻辑函数等类别。利用这些功能，您可以在业务应用程序中创建复杂的数据输入规则等。下面是一些关于如何使用它的想法：

      • 功能齐全的电子表格应用程序
      • 智能文件
      • 教育应用程序
      • 业务逻辑构建器
      • 表格和表格生成器
      • 在线计算器
      • 低连接应用程序

      基本多页示例

      您可以在单页模式下使用公式，也可以在跨页引用的多个 Handsontable 实例中使用公式。

      双击单元格可打开编辑器并预览公式。

        import Handsontable from 'handsontable';
import HyperFormula from 'hyperformula';
import 'handsontable/dist/handsontable.full.min.css';

const data1 = [
  ['10.26', null, 'Sum', '=SUM(A:A)'],
  ['20.12', null, 'Average', '=AVERAGE(A:A)'],
  ['30.01', null, 'Median', '=MEDIAN(A:A)'],
  ['40.29', null, 'MAX', '=MAX(A:A)'],
  ['50.18', null, 'MIN', '=MIN(A1:A5)'],
];

const data2 = [
  ['Is A1 in Sheet1 > 10?', '=IF(Sheet1!A1>10,"TRUE","FALSE")'],
  ['Is A:A in Sheet > 150?', '=IF(SUM(Sheet1!A:A)>150,"TRUE","FALSE")'],
  ['How many blank cells are in the Sheet1?', '=COUNTBLANK(Sheet1!A1:D5)'],
  ['Generate a random number', '=RAND()'],
  ['Number of sheets in this workbook', '=SHEETS()'],
];

// 创建外部超公式实例
const hyperformulaInstance = HyperFormula.buildEmpty({
  // 使用外部 HyperFormula 实例、
  // 使用`'internal-use-in-handsontable'`许可证密钥对其进行初始化
  licenseKey: 'internal-use-in-handsontable',
});

const container1 = document.querySelector('#example-basic-multi-sheet-1');
new Handsontable(container1, {
  data: data1,
  colHeaders: true,
  rowHeaders: true,
  height: 'auto',
  formulas: {
    engine: hyperformulaInstance,
    sheetName: 'Sheet1',
  },
  licenseKey: 'non-commercial-and-evaluation',
});

const container2 = document.querySelector('#example-basic-multi-sheet-2');
new Handsontable(container2, {
  data: data2,
  colHeaders: true,
  rowHeaders: true,
  height: 'auto',
  formulas: {
    engine: hyperformulaInstance,
    sheetName: 'Sheet2',
  },
  licenseKey: 'non-commercial-and-evaluation',
});

          初始化方法

          有多种初始化公式插件的方法。您可以根据自己的使用情况选择最方便的一种。

          在任何情况下，都需要输入 HyperFormula 对象或 HyperFormula 实例：

          import { HyperFormula } from 'hyperformula';

          还有其他安装方法可供选择。查看HyperFormula 的安装文档 (opens new window)

          HyperFormula instance

          要在外部 HyperFormula 实例中使用公式插件，请使用'internal-use-in-handsontable'许可密钥初始化 HyperFormula：

          // 创建外部超公式实例
const hyperformulaInstance = HyperFormula.buildEmpty({
  // 使用"'internal-use-in-handsontable'"许可证密钥对其进行初始化
  licenseKey: 'internal-use-in-handsontable',
});

          将 HyperFormula 类/实例传递给 Handsontable

          {
  formulas: {
    engine: HyperFormula,
    // [插件配置]
  }
}

          or

          {
  formulas: {
    engine: {
      hyperformula: HyperFormula, // or `engine: hyperformulaInstance`
      leapYear1900: false,
      // ......以及更多发动机配置选项。
      // See https://handsontable.github.io/hyperformula/api/interfaces/configparams.html#number
    },
    // [插件配置]
  }
}

          带有外部 HyperFormula 实例的单一 Handsontable 实例

          const hyperformulaInstance = HyperFormula.buildEmpty({
  // 使用外部 HyperFormula 实例、
  // 使用`'internal-use-in-handsontable'`许可证密钥对其进行初始化
  licenseKey: 'internal-use-in-handsontable',

{

    engine: hyperformulaInstance;
  }
}

          可用功能

          本插件继承了 HyperFormula 的计算功能。完整的函数参考资料可在 HyperFormula 文档 (opens new window)中找到。

          afterFormulasValuesUpdate 钩子

          此钩子侦听计算引擎中单元格的任何更改，包括包含公式的从属单元格。

          const afterFormulasValuesUpdate = (changes) => {
  changes.forEach((change) => {
    console.log('change', change.address, change.newValue);
  });
};

new Handsontable(element, {
  formulas: {
    engine: HyperFormula,
  },
  afterFormulasValuesUpdate,
});

          命名表达式

          您可以在公式表达式中使用自定义命名表达式。命名表达式既可以是普通值，也可以是带有绝对单元格地址引用的公式。要注册命名表达式，请将包含名称和表达式的数组传递给公式配置对象： 

            import Handsontable from 'handsontable';
import HyperFormula from 'hyperformula';
import 'handsontable/dist/handsontable.full.min.css';

const data = [
  ['Travel ID', 'Destination', 'Base price', 'Price with extra cost'],
  ['154', 'Rome', 400, '=ROUND(ADDITIONAL_COST+C2,0)'],
  ['155', 'Athens', 300, '=ROUND(ADDITIONAL_COST+C3,0)'],
  ['156', 'Warsaw', 150, '=ROUND(ADDITIONAL_COST+C4,0)'],
];

const container = document.querySelector('#example-named-expressions1');
const hotNamedExpressions = new Handsontable(container, {
  data: data,
  colHeaders: true,
  rowHeaders: true,
  height: 'auto',
  formulas: {
    engine: HyperFormula,
    namedExpressions: [
      {
        name: 'ADDITIONAL_COST',
        expression: 100,
      },
    ],
  },
  licenseKey: 'non-commercial-and-evaluation',
});

const input = document.getElementById('named-expressions-input');
const formulasPlugin = hotNamedExpressions.getPlugin('formulas');
const button = document.getElementById('named-expressions-button');

button.addEventListener('click', (event) => {
  formulasPlugin.engine.changeNamedExpression('ADDITIONAL_COST', input.value);
  hotNamedExpressions.render();
});

            已知限制

            • 不支持使用IndexMapperAPI 以编程方式移动包含公式的行或列。请使用ManualRowMove或ManualColumnMoveAPI。
            • 公式不支持getSourceData()，因为该方法对源数据（使用物理索引）进行操作，而公式对可视化数据（使用可视化索引）进行操作。
            • 公式不支持嵌套数据，即当 Handsontable 的数据设置为嵌套对象数组时。

            支持 HyperFormula 版本

            不同版本的 Handsontable 支持不同版本的 HyperFormula。

            要了解使用哪个版本的 HyperFormula，请参阅下表：

            Handsontable version HyperFormula version
            8.x.x (opens new window) and lower No HyperFormula support (old Formulas (opens new window) plugin)
            9.x.x (opens new window) 0.6.2 (opens new window)
            10.x.x (opens new window) ^1.2.0 (opens new window)
            11.x.x (opens new window) ^1.2.0 (opens new window)
            12.x.x (opens new window) and higher ^2.0.0 (opens new window)

            TIP

            您只能在连接到 Handsontable 实例的 HyperFormula 实例中使用'internal-use-in-handsontable'许可证密钥。

            要在 Handsontable 实例之外使用 HyperFormula（例如在服务器上），您需要一个专用的[HyperFormula 许可证密钥(https://hyperformula.handsontable.com/guide/license-key.html)。有关详细信息，请联系我们的销售团队 (opens new window)。
