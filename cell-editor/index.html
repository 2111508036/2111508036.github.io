      概述

      Handsontable 将显示单元格值的过程与更改值的过程分开。渲染器负责显示数据，而编辑器负责更改数据。由于渲染器只有一个简单的任务：获取单元格的实际值并将其表示为 HTML 代码返回，因此它们可以是一个单独的函数。然而，编辑器需要处理用户输入（即鼠标和键盘事件）、验证数据并根据验证结果进行操作，因此将所有这些功能放在一个函数中并不是一个好主意。这就是 Handsontable 编辑器由编辑器类表示的原因。

      本教程将让您全面了解单元格编辑的整个过程、Handsontable核心如何管理编辑器、编辑器的生命周期以及如何创建自己的编辑器。

      EditorManager是一个负责处理 Handsontable 中所有可用编辑器的类。如果 Handsontable 需要与编辑器交互，则会使用EditorManager对象。EditorManager对象在init()方法中实例化，该方法在首次调用 Handsontable() 构造函数后运行。在 Handsontable 实例中，EditorManager对象的引用是私有的，你无法访问它。不过，有一些方法可以改变EditorManager的默认行为，稍后将详细介绍。

      编辑器管理器任务

      EditorManager有 4 项主要任务:

      • 为活动单元格选择合适的编辑器
      • 准备显示编辑器
      • 显示编辑器（基于用户行为）
      • 关闭编辑器（基于用户行为）

      我们将详细讨论其中的每一项任务。

      为活动单元格选择合适的编辑器

      当用户选择一个单元格时，EditorManager通过检查编辑器配置选项的值，找到分配给该单元格的编辑器类。您可以全局（针对表格中的所有单元格）、每列（针对列中的所有单元格）或单独针对每个单元格定义编辑器配置选项。更多详情，请参阅配置选项指南。

      编辑器配置选项的值可以是代表编辑器的字符串（如 "文本"、"自动完成"、"复选框 "等），也可以是编辑器类。然后，EditorManager 将获得一个编辑器类的实例，首先要记住的重要一点是：在一个表中总是有一个编辑器类的实例，换句话说，每个编辑器类对象在一个表中都是单例，这意味着每个表只调用一次它的构造函数。如果一个页面上有 3 个表，那么每个表都会有自己的编辑器类实例。这有一些重要的影响，你必须考虑创建自己的编辑器。

      准备显示编辑器

      编辑器管理器获取编辑器类实例（编辑器对象）时，会调用其prepare()方法。prepare()方法会设置与所选单元格相关的编辑器对象属性，但不会显示编辑器。每次用户选择单元格时，都会调用prepare()方法。在某些情况下，同一个单元格可以多次调用prepare()，而不会改变选择。

      显示屏编辑器

      编辑器准备就绪后，EditorManager会等待触发单元格编辑的用户事件。这些事件包括

      • 按Enter
      • 按Shift + Enter
      • 双击单元格
      • 按F2

      如果这些事件中的任何一个被触发，EditorManager就会调用编辑器的beginEditing()方法，从而显示编辑器。

      关闭编辑器

      编辑器打开后，EditorManager会等待应结束单元格编辑的用户事件。这些事件包括

      • 单击另一个单元格（保存更改）
      • 按Enter (保存更改并将选择向下移动一格)
      • 按Shift + Enter (保存更改并将选择向上移动一格)
      • 按Ctrl/Cmd + Enter or Alt/Option + Enter (在单元格内添加新行)
      • 按Escape (中止更改)
      • 按Tab (保存更改，并根据布局方向将一个单元格向右或向左移动)
      • Pressing Shift + Tab (保存更改，并根据布局方向将一个单元格向左或向右移动)
      • Pressing Page Up, Page Down (保存更改并向上/向下移动一个屏幕)

      如果触发了这些事件中的任何一个，EditorManager会调用编辑器的finishEditing()方法，该方法应尝试保存更改（除非按下了 ESC 键）并关闭编辑器。

      重写 EditorManager 的默认行为

      您可能希望更改导致编辑器打开或关闭的默认事件。例如，您的编辑器可能会使用Arrow Up和Arrow Down事件来执行某些操作（如增加或减少单元格值），而您不希望 EditorManager在用户按下这些键时关闭编辑器。这就是EditorManager在处理用户事件前运行beforeKeyDown钩子的原因。如果为 beforeKeyDown注册一个监听器，那么在事件对象上调用stopImmediatePropagation()时，EditorManager将执行其默认操作。有关覆盖EditorManager行为的更多信息，请参阅 "SelectEditor - 从零开始创建编辑器 "一节。

      现在你应该对EditorManager的工作原理有了更好的了解。让我们再深入一点，看看每个编辑器类都必须实现哪些方法以及这些方法的作用。

      BaseEditor

      Handsontable.editors.BaseEditor是一个抽象类，所有编辑器类都应从该类继承。它实现了一些基本的编辑器方法，并声明了一些每个编辑器类都应实现的方法。在本节中，我们将研究所有这些方法。

      常用方法

      常用方法是由BaseEditor类实现的方法。它们包含每个编辑器都应具备的一些核心逻辑。大多数情况下，您不需要理会这些方法。不过，如果你要创建一些更复杂的编辑器，你可能会想覆盖一些常用方法，在这种情况下，你应该总是调用原始方法，然后执行其他操作，具体到你的编辑器。

      示例 - 覆盖常用方法

      // CustomEditor 是一个类，继承自 BaseEditor
class CustomEditor extends BaseEditor {
  prepare(row, col, prop, td, originalValue, cellProperties) {
    // 调用原始方法...
    super.prepare(row, col, prop, td, originalValue, cellProperties);
    // ......然后执行一些与自定义编辑器相关的操作
    this.customEditorSpecificProperty = 'foo';
  }
}

      常见的方法有 7 种。下面将一一介绍。

      prepare(row: Number, col: Number, prop: Number|String, td: HTMLTableCellElement, originalValue: Mixed, cellProperties: Object)

      准备为给定单元格显示编辑器。设置大部分实例属性。

      返回: undefined

      beginEditing(newInitialValue: Mixed, event: Mixed)

      将编辑器值设置为newInitialValue。如果newInitialValue未定义，编辑器值将设置为原始单元格值。内部调用open()方法。

      返回: undefined

      finishEditing(restoreOriginalValue: 'Boolean' [optional], ctrlDown: Boolean [optional], callback: Function)

      尝试完成单元格编辑。内部调用saveValue()和discardEditor()。如果restoreOriginalValue设置为true，单元格值将被设置为其原始值（编辑前的值）。ctrlDown值将作为第二个参数传递给saveValue()。

      回调函数包含一个布尔参数--如果新值有效或allowInvalid配置选项设置为true，否则该参数为false。

      discardEditor(result: Boolean)

      单元格验证结束时调用。如果新值保存成功（结果设置为true或allowInvalid属性为true），则调用close()方法，否则调用focus()方法并保持编辑器打开状态。

      返回: undefined

      saveValue(value: Mixed, ctrlDown: Boolean)

      尝试将值保存为新的单元格值。内部执行验证。如果ctrlDown设置为"true"，新值将被设置为所有选定单元格的值。

      返回: undefined

      isOpened()

      如果编辑器已打开，则返回true；如果编辑器已关闭，则返回false。调用open()方法后，编辑器被视为已打开。调用close()方法后，编辑器被视为关闭。

      返回: Boolean

      extend()

      返回Function - 从当前类继承的类函数。可以安全地覆盖返回类的原型方法，而不会改变父类的原型。

      示例 - 继承自BaseEditor并重写其方法

      const CustomEditor = Handsontable.editors.BaseEditor.prototype.extend();

// 这不会改变 BaseEditor.prototype.beginEditing()
CustomEditor.prototype.beginEditing = function() {};

      示例 - 从另一个编辑器继承

      const CustomTextEditor = Handsontable.editors.TextEditor.prototype.extend();

// CustomTextEditor 使用 TextEditor 实现的所有方法。
// 您可以安全地覆盖任何方法，而不会影响原始 TextEditor。

      注: 这是一种实用方法，与编辑单元格的过程无关。

      编辑器专用方法

      编辑器专用方法是BaseEditor中未实现的方法。为了正常工作，每个编辑器类都必须实现这些方法。

      init()

      当创建编辑器类的新实例时调用的方法。这在每个表实例中最多发生一次，因为所有使用的编辑器都是表实例中的单子。您应该使用该方法来执行一些任务，这些任务的结果可以在编辑器的生命周期内重复使用。最常见的操作是创建编辑器的 HTML 结构。

      方法无需返回任何值。

      getValue()

      方法应返回当前编辑器值，即应保存为新单元格值的值。

      setValue(newValue: Mixed)

      方法应将编辑器值设置为newValue.

      示例 假设我们正在实现一个DateEditor，它通过显示日历来帮助选择日期。getvalue()和setvalue()方法可以这样工作：

      class CalendarEditor extends TextEditor {
  constructor(hotInstance) {
    super(hotInstance);
  }

  getValue() {
    // 返回当前选择的日期，例如 "2023/09/15"
    return calendar.getDate();
  }

  setValue(newValue) {
    // 突出显示日历上的指定日期
    calendar.highlightDate(newValue);
  }
}

      open()

      显示编辑器。在大多数情况下，这种方法可以很简单：

      class CustomEditor extends TextEditor {
  open() {
    this.editorDiv.style.display = '';
  }
}

      该方法无需返回任何值。

      close()

      更改单元格值后隐藏编辑器。在大多数情况下，这种方法可以非常简单：

      class CustomEditor extends TextEditor {
  close() {
    this.editorDiv.style.display = 'none';
  }
}

      该方法无需返回任何值。

      focus()

      聚焦编辑器。当用户想通过选择另一个单元格来关闭编辑器，而编辑器中的值没有生效（且allowInvalid为false）时，就会调用该方法。在大多数情况下，该方法可以像以下方法一样简单： 

      class CustomEditor extends TextEditor {
  focus() {
    this.editorInput.focus();
  }
}

      该方法无需返回任何值。

      常用编辑器属性

      编辑器实例可通过 this 对象（如this.instance）获得上述所有属性。

      性质 类型 说明
      instance Handsontable.Core 编辑器对象所属的 Handsontable 实例。在类构造函数中设置，在编辑器的整个生命周期中不可更改。
      row Number 活动单元格行索引。在每次调用prepare()方法时更新。
      col Number 活动单元格列索引。在每次调用prepare()方法时更新。
      prop String 与活动单元格相关的属性名称（仅当数据源是对象数组时才相关）。在每次调用prepare()方法时更新。
      TD HTMLTableCellNode 活动单元格的节点对象。在每次调用prepare()方法时更新。
      cellProperties Object 表示活动单元格属性的对象。在每次调用prepare()方法时更新。

      如何创建自定义编辑器？

      现在你已经了解了 Handsontable 编辑器背后的理念，并准备好编写自己的编辑器了。基本上，你可以通过创建一个继承自BaseEditor的新编辑器类，从头开始创建一个新的编辑器；如果你只想增强现有的编辑器，也可以扩展它的类，只覆盖它的一些方法。

      在本教程中，我们将研究这两种方法。我们将创建一个全新的 SelectEditor，它使用<select>列表来更改单元格的值。我们还将创建一个 PasswordEditor，它的工作方式与常规的TextEditor完全相同，只是它显示的是密码输入而不是文本区域。

      让我们从密码编辑器开始，因为它比较简单。

      PasswordEditor - 扩展现有编辑器

      TextEditor 是 Handsontable 默认提供的最复杂的编辑器。它会显示一个 <textarea>，并自动改变其大小以适应其内容。我们希望创建一个 PasswordEditor，保留所有这些功能，但显示 <input type="password"> 字段而不是 <textarea>。

      您可能已经猜到了，我们需要创建一个继承自 TextEditor 的新编辑器类，然后覆盖它的某些方法，用 input:password 替换 <textarea> 。幸运的是，textarea 和密码输入具有相同的 API，因此我们只需替换负责创建 HTML 元素的代码即可。如果你查看一下 TextEditor init() 方法，就会发现它调用了内部 createElements() 方法，该方法会创建 <textarea> 节点，并在编辑器初始化时将其添加到 DOM 中--BINGO！

      代码如下

      import Handsontable from 'handsontable';

class PasswordEditor extends Handsontable.editors.TextEditor {
  createElements() {
    super.createElements();

    this.TEXTAREA = this.hot.rootDocument.createElement('input');
    this.TEXTAREA.setAttribute('type', 'password');
    this.TEXTAREA.setAttribute('data-hot-input', true); // 使 HOT 可以将该元素识别为其自身组件的元素。
    this.textareaStyle = this.TEXTAREA.style;
    this.textareaStyle.width = 0;
    this.textareaStyle.height = 0;

    this.TEXTAREA_PARENT.innerText = '';
    this.TEXTAREA_PARENT.appendChild(this.TEXTAREA);
  }
}

      就是这样！现在您可以使用新的编辑器了：

      const container = document.querySelector('#container')
const hot = new Handsontable(container, {
  columns: [
    {
      type: 'text'
    },
    {
      editor: PasswordEditor
      // 如果要使用字符串'password'，而不是通过
      // 实际的编辑器类请参阅 "注册编辑器 "部分
    }
  ]
});

      让我们尝试更复杂的方法：从头开始创建一个新的编辑器。

      SelectEditor - 从零开始创建编辑器

      我们将创建一个功能齐全的编辑器，让用户使用标准 <select> 输入法从预定义的选项列表中选择单元格值。作为额外功能，我们将添加使用 ARROW_UP 和 ARROW_DOWN 键更改当前选定选项的功能。

      可做的事:

      1. 创建一个继承自Handsontable.editors.BaseEditor的新类.
      2. 添加创建<select>输入并附加到 DOM 的函数.
      3. 添加使用单元格属性中传递的选项数组填充 <select> 的函数.
      4. 实施方法：
        • getvalue()
        • setvalue()
        • open()
        • close()
        • focus()
      5. 覆盖EditorManager的默认行为，使按Arrow Up键和Arrow Down键不会关闭编辑器，而是更改当前选中的值。
      6. 注册编辑器.

      创建新编辑器

      这可能是最简单的部分。我们只需调用BaseEditor.prototype.extend()函数，它将返回一个继承自BaseEditor的新函数类。

      const SelectEditor = Handsontable.editors.BaseEditor.prototype.extend();

      任务一: 完成

      创建 <select> 输入并将其附加到 DOM

      我们可以将创建 <select> 元素并将其放入 DOM 的函数放在三个可能的位置：

      • init() 方法
      • prepare() 方法
      • open() 方法

      选择最佳解决方案的关键在于了解这些方法中的每种方法何时被调用。

      init()方法在创建编辑器类对象时被调用。这种情况在每个表实例中最多发生一次，因为一旦创建了对象，EditorManager每次询问该编辑器类实例时都会重复使用该对象（详见单例模式 (opens new window)）。

      prepare()方法，每次用户选择将此特定编辑器类设置为编辑器配置选项的单元格时，该方法都会被调用。因此，如果我们将SelectEditor设置为一整列的编辑器，那么选择这一列中的任何单元格都会调用SelectEditor的prepare()方法。换句话说，这个方法在表格运行期间会被调用数百次，尤其是在处理大型数据时。prepare()方法的另一个重要方面是，它不应显示编辑器（这是open的工作）。显示编辑器是由用户事件触发的，例如按ENTER 键、F2 键或双击单元格，因此调用prepare()和实际显示编辑器之间会有一些时间间隔。不过，prepare()执行的操作应尽快完成，以提供最佳的用户体验。

      open()方法在需要显示编辑器时被调用。在大多数情况下，该方法应将 CSS display属性更改为 block 或执行类似操作。用户希望在触发事件（按下适当的键或双击单元格）后立即显示编辑器，因此open()方法应尽可能快地工作。

      综上所述，将负责创建<select>输入的代码放在init()方法中的某个位置最为合理。DOM 操作被认为是相当昂贵（关于资源消耗）的操作，因此最好只执行一次，并在编辑器的整个生命周期中重复使用生成的 HTML 节点。

      import Handsontable from 'handsontable';

class SelectEditor extends Handsontable.editors.BaseEditor {
  /**
   * 初始化编辑器实例、DOM 元素和挂载钩子。
   */
  init() {
    // 创建分离节点，添加 CSS 类并确保其不可见
    this.select = this.hot.rootDocument.createElement('SELECT');
    this.select.classList.add('htSelectEditor');
    this.select.style.display = 'none';

    // 将节点附加到保存表格的容器中，从而将节点附加到 DOM 中
    this.hot.rootElement.appendChild(this.select);
  }
}

      .htSelectEditor {
  /*
   * 此黑客功能可在 WebKit 浏览器中更改 <select> 尺寸
   */
  -webkit-appearance: menulist-button !important;
  position: absolute;
  width: auto;
  z-index: 300;
}

      任务二: 完成

      用选项填充

      在上一步中，我们实现了一个创建<select>输入并将其附加到 DOM 的函数。您可能注意到，我们没有编写任何创建<option>元素的代码，因此如果我们显示列表，它将是空的。

      我们希望能够这样定义一个选项列表：

      const container = document.querySelector('#container')
const hot = new Handsontable(container, {
  columns: [
    {
      editor: SelectEditor,
      selectOptions: ['option1', 'option2', 'option3']
    }
  ]
});

      我们无法（轻松）获取selectOptions的值。即使我们能获取该数组的值，也只能在"init"函数中为列表填充一次选项。如果我们有不止一列使用SelectEditor，而且每一列都有自己的选项列表，该怎么办？甚至有可能同一列中的两个单元格拥有不同的选项列表（级联配置--还记得吗？ 显然，我们必须为创建列表项的代码找到一个更好的位置。

      我们只剩下prepare()和open()两种方法。后者实现起来更简单，但正如我们之前所说，setvalue()应该尽可能快地工作，如果selectOptions包含很长的选项列表，那么创建<option>节点并将其附加到 DOM 可能会很耗时。因此，prepare()似乎是进行此类工作的更安全的地方。唯一需要注意的是，在重载prepare()时，我们应始终调用BaseEditor的原始方法。BaseEditor.prototype.prepare()设置了一些重要属性，这些属性会被其他编辑器方法使用。 

      // 在 prepare() 方法中创建选项
prepare(row, col, prop, td, originalValue, cellProperties) {
  // 记住要调用父代的方法
  super.prepare(row, col, prop, td, originalValue, cellProperties);

  const selectOptions = this.cellProperties.selectOptions;
  let options;

  if (typeof selectOptions === 'function') {
    options = this.prepareOptions(selectOptions(this.row, this.col, this.prop));
  } else {
    options = this.prepareOptions(selectOptions);
  }

  this.select.innerText = '';

  Object.keys(options).forEach((key) => {
    const optionElement = this.hot.rootDocument.createElement('OPTION');
    optionElement.value = key;
    optionElement.innerText = options[key];
    this.select.appendChild(optionElement);
  });
}

      prepareOptions的位置:

      prepareOptions(optionsToPrepare) {
  let preparedOptions = {};

  if (Array.isArray(optionsToPrepare)) {
    for (let i = 0, len = optionsToPrepare.length; i < len; i++) {
      preparedOptions[optionsToPrepare[i]] = optionsToPrepare[i];
    }

  } else if (typeof optionsToPrepare === 'object') {
    preparedOptions = optionsToPrepare;
  }

  return preparedOptions;
}

      getvalue()、setvalue()和close()的实现方法不言自明，但open()的实现方法需要稍加说明。首先，该实现假定负责将选项填充到列表中的代码是放在prepare()中的。其次，在显示列表之前，我们会设置列表的高度和最小宽度，使其与相应单元格的大小相匹配。这是一个可选步骤，但如果不这样做，编辑器的大小就会因浏览器而异。也许限制<select>的最大高度也是一个好主意。最后，由于<select>已添加到表格容器的末尾，我们必须改变其位置，使其显示在正在编辑的单元格上方。这也是一个可选步骤，但将编辑器放在相应单元格的旁边似乎非常合理。

      使用Arrow Up和Arrow Down更改所选值

      我们知道我们的编辑器可以正常工作，但让我们再对它进行一次调整。目前，当编辑器打开后，用户按下Arrow Up或Arrow Down箭头键，编辑器就会关闭，选择的单元格会向上或向下移动一个。如果按向上和向下箭头键能改变当前选中的值，那岂不是更好？用户可以导航到单元格，按Enter键，选择想要的值，然后再次按Enter键保存更改。这样就可以在不使用鼠标的情况下处理表格了。听起来不错，但如何覆盖默认行为呢？毕竟，何时关闭编辑器是由`编辑器管理器决定的。

      别担心。虽然您无法直接访问EditorManager实例，但仍可重写其行为。在EditorManager开始处理键盘事件之前，它会触发beforeKeyDown钩子。如果任何监听函数在事件对象上调用stopImmediatePropagation()方法，EditorManager将不再处理该事件。因此，我们只需注册一个beforeKeyDown监听函数，检查是否按下了Arrow Up或Arrow Down，如果是，则停止事件传播，并相应更改 <select>列表中的当前选中值。

      我们需要牢记的是，我们的监听器只能在编辑器打开时工作。我们希望为其他编辑器以及未打开编辑器的编辑器保留默认行为。因此，最合理的监听器注册位置是open()方法，而 close()方法应包含移除监听器的代码。

      下面是监听器函数的样子：

      onBeforeKeyDown() {
  const previousOptionIndex = this.select.selectedIndex - 1;
  const nextOptionIndex = this.select.selectedIndex + 1;

  switch (event.keyCode) {
    case 38: // 向上箭头
      if (previousOptionIndex >= 0) {
        this.select[previousOptionIndex].selected = true;
      }

      event.stopImmediatePropagation();
      event.preventDefault();
      break;

    case 40: // 向下箭头
      if (nextOptionIndex <= this.select.length - 1){
        this.select[nextOptionIndex].selected=true;
      }

      event.stopImmediatePropagation();
      event.preventDefault();
      break;

    default:
      break;
  }
}

      活动编辑器是最近调用prepare()方法的编辑器。例如，如果您选择了一个编辑器为Handsontable.TextEditor的单元格，那么getActiveEditor()将返回该编辑器类的一个对象。如果再选择一个编辑器为Handsontable.DateEditor的单元格（可能是另一列中的单元格），则活动编辑器将发生变化，此时 getActiveEditor() 将返回DateEditor类对象。

      代码的其余部分应该很清楚了。现在我们要做的就是注册我们的监听器。

      open() {
  this.addHook('beforeKeyDown', () => this.onBeforeKeyDown());
}

close() {
  this.clearHooks();
}

      注册编辑器

      当你创建一个编辑器时，一个好主意是给它分配一个别名，用来指代这个特定的编辑器类。Handsontable 默认定义了 11 个别名：

      • autocomplete for Handsontable.editors.AutocompleteEditor
      • base for Handsontable.editors.BaseEditor
      • checkbox for Handsontable.editors.CheckboxEditor
      • date for Handsontable.editors.DateEditor
      • dropdown for Handsontable.editors.DropdownEditor
      • handsontable for Handsontable.editors.HandsontableEditor
      • numeric for Handsontable.editors.NumericEditor
      • password for Handsontable.editors.PasswordEditor
      • select for Handsontable.editors.SelectEditor
      • text for Handsontable.editors.TextEditor
      • time for Handsontable.editors.TimeEditor

      它为用户提供了一种方便的方法，用于定义在更改某些单元格的值时应该使用哪个编辑器。用户无需知道哪个类负责显示编辑器，甚至无需知道是否存在任何类。此外，您还可以更改与别名相关的类，而无需更改定义表格的代码。

      要注册自己的别名，请使用Handsontable.editors.registerEditor()函数。它需要两个参数：

      • editorName - 表示编辑器的字符串
      • editorClass - 将由editorName表示的类

      如果您想在别名 select 下注册 SelectEditor，则必须调用：

      Handsontable.editors.registerEditor('select', SelectEditor);

      明智选择别名。如果用已注册的名称注册编辑器，目标类将被覆盖：

      Handsontable.editors.registerEditor('text', MyNewTextEditor);

      现在，"text"别名指向 MyNewTextEditor 类，而不是Handsontable.editors.TextEditor.

      因此，除非您有意覆盖现有别名，否则请尽量选择一个唯一的名称。一个好的做法是在别名前加上一些自定义名称（例如你的 GitHub 用户名），以尽量减少名称冲突的可能性。如果你想发布你的编辑器，这一点尤为重要，因为你永远不知道使用你编辑器的用户注册了别名。

      Handsontable.editors.registerEditor('select', SelectEditor);

      可能已经有人注册了这样的别名。

      Handsontable.editors.registerEditor('my.select', SelectEditor);

      这样好多了。

      准备发布编辑器

      如果您打算发布您的编辑器，或者只是想让您的代码保持整洁漂亮（我们都是这样做的😃），有 3 个简单的步骤可以帮助您整理您的代码。

      附在IIFE中

      将代码放在模块中，避免污染全局命名空间。您可以使用 AMD、CommonJS 或其他任何模块模式，但隔离代码的最简单方法是使用普通的立即调用函数表达式（IIFE）。

      (Handsontable => {
  const CustomEditor = Handsontable.editors.BaseEditor.prototype.extend();

  // ...编辑器的其他代码

})(Handsontable);

      将Handsontable命名空间作为参数传递是可选的（因为它是全局定义的），但使用尽可能少的全局对象是一种很好的做法，这样可以使模块化和依赖关系管理更容易。

      将编辑器添加到专用命名空间

      用 IIFE 封装的代码无法从外部访问，除非它被有意暴露。为了使事情井井有条，请使用Handsontable.editors.registerEditor方法将您的编辑器注册到编辑器集合中。这样，您就可以在定义表格时使用您的编辑器，而其他用户也可以方便地访问您的编辑器，以备扩展之用。 

      (Handsontable => {
  const CustomEditor = Handsontable.editors.BaseEditor.prototype.extend();

  // ...编辑器的其他代码

  // 最后
  Handsontable.editors.registerEditor('custom', CustomEditor);

})(Handsontable);

      从现在起，您可以这样使用CustomEditor：

      const container = document.querySelector('#container');
const hot = new Handsontable(container, {
  columns: [{
    editor: Handsontable.editors.CustomEditor
  }]
});

      扩展CustomEditor也很容易。

      const AnotherEditor = Handsontable.editors.getEditor('custom').prototype.extend();

      请记住，您选择的名称没有限制，但要明智选择，不要覆盖现有的编辑器。尽量保持名称的唯一性。

      注册别名

      最后一步是将编辑器注册为别名，这样用户就可以轻松引用编辑器，而无需输入实际的类名。详见注册编辑器。

      总之，精心准备的编辑应该是这样的：

      (Handsontable => {
  const CustomEditor = Handsontable.editors.BaseEditor.prototype.extend();

  // ...编辑器的其他代码

  // 将编辑器放入专用命名空间
  Handsontable.editors.CustomEditor = CustomEditor;

  // 注册别名
  Handsontable.editors.registerEditor('theBestEditor', CustomEditor);

})(Handsontable);

      从现在起，您可以这样使用CustomEditor：

      const container = document.querySelector('#container')
const hot = new Handsontable(container, {
  columns: [{
    editor: 'theBestEditor'
  }]
});
